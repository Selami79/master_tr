Organizing Control Definition Defaults

It is important to realize that when you are working in the Control Definition Manager, you are seeing data from two different places:

  • Post text, miscellaneous integer and real variables are read from the .pst file. You can see this data on the Misc int/real page and the Text pages. The Control Definition Manager loads these from the .pst file when you select a post processor.
  • Values and settings from all other control definition data pages are read from the .mcam-control file.

Therefore, when you create a new control definition, Mastercam gets the default values from two different places:

  • Default post text is read from the .pst file.
  • Defaults for other control definition pages are read from the .mcam-control file. Each type of default is described below.
NOTE

The Control Definition Manager also includes an Operation Defaults page that lets you work with toolpath and operation defaults. These defaults are stored separately in a .mcam-defaults file, and are not part of this topic.

Default values for regular control definition pages

For regular control definition data pages, there are two layers of defaults:

  • Mastercam maintains some basic system "factory" defaults. When you create a completely new, blank control file, Mastercam uses the factory defaults. You can access or customize some factory defaults in the System Configuration dialog box.
  • Inside each control file, you can create an additional layer of defaults for each control type. You can create, review, and edit them when you see Default settings for control type as the post selection in the Control Definition Manager. These defaults do not exist unless you create and save them, and they only apply to new control definitions created inside the same control file.

Use the right-click menu in the Control Definition Manager to reset the values on a page to the default values. When you right-click in a data page and choose Default, Mastercam automatically resets the values to the defaults stored in the current control file, if it can find them. Otherwise, it will reset the data to the system defaults.

Default values for post text

Default values for post text and miscellaneous values are managed separately from the other control definition defaults. These are stored in each .pst file under a [DEFAULT] header. Separate headers let you maintain defaults for each control type. Use the Text page in the Control Definition Manager to create and manage post text defaults.

Mastercam also maintains a set of basic system factory defaults for post text and miscellaneous values that it can reference when post-specific defaults are not available or when you choose not to use them.

Use the right-click menu on the Text page of the Control Definition Manager to access default values for post text or to reset the post text to default values. You can do this for either an entire Text page using All sheets, or an individual category or cell within the page. When you reset the post text to default values, you can choose to reset it to either the basic factory defaults (System), or to read the [DEFAULT] header from the .pst file (Post). If there is no [DEFAULT] header, the system defaults are used.

If you choose to Save as default, Mastercam writes the current values to the [DEFAULT] section of the current .pst file for the currently selected control type.

You can also choose to Restore post text values.

In this case, the values that get imported depend on the post state.

  • If the .pst file already has a post text section — and it includes an entry for the desired category, sheet, or cell — Mastercam will read that entry into the current Text page.
  • Otherwise, it will read a value from the [DEFAULT] section of the .pst file.
  • If it can't find appropriate values there, it will restore from the system defaults.
