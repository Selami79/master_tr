How Mastercam Organizes Control Definitions

Mastercam uses control files to store and organize control definitions. These are organized by machine type and post processor.

Each control definition consists of a machine type linked to a specific post processor. Each control definition inside a control file has a different combination of machine type and post processor. For example, a typical control file might look like this:

FANUC.mcam-control Control definition #1: MILL + myfanuc.pst Control definition #2: ROUTER + myfanuc.pst Control definition #4: LATHE + fanuc_lathe.pst Control definition #6: LATHE + fanuc_vtl.pst Control definition #7: MILL + fanuc5ax.pst and so on.

When you are building a machine definition for a machine that uses this control and select FANUC.mcam-control in the Machine Definition Manager, you will then see the list of control definitions as shown above so that you can select the one you want for the machine. The Machine Definition Manager will only show you control definitions for the machine type you are creating.

For each control file:

  • The set of control types is fixed: Mill, Lathe, Router, Mill-Turn, or Wire
  • The posts that are available are determined by you. To make a post available, click Post processors and choose Add files.

When you are working in the Control Definition Manager, you can see which control definition is loaded by looking at these two fields:

Clicking Existing definitions to see which control definitions are stored in the current control file. Click on a definition to load it in the Control Definition Manager.

Many users will find it easiest to have one control definition per .mcam-control file. You can accomplish this by using the Save As button whenever you change the control type or select a different post processor, and saving to a different control file. Otherwise, Mastercam will add a new control definition to the current control file.

NOTE

The control definition that gets loaded into the machine group and saved with the part file is only the individual control definition for a single machine type-post combination, not the entire contents of the control file.

Default control definitions

Mastercam also lets you create a set of default values for each machine type and store them in the control file. This is like a control definition with no post processor selected. When you are in the Control Definition Manager, it looks like this:

Each set of default values is stored in the control file as a separate control definition. For example, in the case of the sample FANUC.mcam-control mentioned above:

Control definition #8: MILL Default settings Control definition #9: ROUTER Default settings Control definition #10: LATHE Default settings

These cannot be selected in the machine definition or used for posting, but you can use them as a generic template for creating specific control definitions. For example, before creating a specific control definition, you might follow these steps:

  • Load a default control definition.
  • Select a specific post.
  • Create or modify the post text and miscellaneous values.
  • Edit only the other settings that need to be changed from the defaults.
  • Save the new control definition. Choose whether to save it in a new control file or in the same control file as the defaults.
