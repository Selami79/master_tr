Introduction to Mill-Turn Steady Rests

Mastercam Mill-Turn supports a wide range of steady rest applications. Steady rest operations are divided into two types:

  • Steady rest operations are used by standalone steady rests. These are mounted to the machine base. Standalone steady rest components are created by licensed machine developers and are stored in the machine definition. Multiple steady rests are supported: each .machine file can include up to eight steady rests.

  • Steady rest point operations are used by turret-mounted steady rests. These are selected and mounted by the user during Job Setup. Turret mounted steady rest components can be created by any Mill-Turn user and are stored in component libraries. This lets you import them from another library or share them between different machines. You can store as many turret mounted steady rest components as you like in a component library.

Each steady rest component regardless of type is represented by solid models and fully simulated. Arms and pistons are represented as individual subcomponents so they can be configured and simulated as accurately as possible. See Adding a Turret Mounted Steady Rest to a Component Library (Mill-Turn) to learn more.

Once the steady rest components have been created, users choose which steady rest to use and configure it as part of the Job Setup process. See Selecting a Steady Rest: Mill-Turn Job Setup to learn more.

Mastercam lets you program steady rests in a number of different ways.

  • Build steady rest operations by assembling individual events such as linear or rapid positioning moves and clamping/unclamping actions so you can customize the operations for your machine or application. This also lets you create syncs between different steady rest events and your other toolpaths or part handling operations.

  • You can create steady rest operations that follow the tool or other turret or that follow the subspindle home after a part transfer operation.

  • Mastercam supports multiple standalone or machine-mounted steady rests in a single .machine file. These can be programmed individually or linked together so that when one steady rest moves, the other moves in tandem with it.

  • Steady rest events can be incorporated into Mill-Turn's part handling strategies and created automatically by selecting the desired strategy.

See Creating a Steady Rest Operation: Mill-Turn to learn more.

NOTE

Mastercam Lathe and Mill-Turn support steady rests differently. Each uses different steady rest components and operations. See Lathe Steady Rest dialog box to learn about steady rest support in Lathe.
