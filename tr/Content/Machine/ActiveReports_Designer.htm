ActiveReports Designer

ActiveReports Designer is a third-party application that allows you to customize the content and layout of any ActiveReports template. With this application, you can modify an existing template or create an entirely new one. ActiveReports Designer is located in the ..\common\reports directory of the Mastercam installation. To start the application, double-click ActiveReports_Designer.exe.

An ActiveReport template calls on information retrieved by an XML file from data generated by Mastercam at run-time. While all ActiveReports templates reside in folders in the ..\<Shared Data Folder>\common\reports directory, the exact default location of each template depends on the report type. The default location of a report's XML source is in the XML directory in ..\<Shared Data Folder>\common\reports.

Using subreports

An ActiveReports template creates a report with single pieces of data obtained directly from the XML file and/or sets of data that are obtained from the XML file via a subreport. A "parent" template uses a subreport to funnel multiple instances of the same data type into a report. Typically, an ActiveReports template for Mastercam would use subreports to create a section for each selected tool or operation.

  1. Mastercam data
  2. XML file
  3. Subreport
  4. Report template
  5. Generated report

A subreport is simply a template file (RPX) that is associated with a subreport control in the ActiveReports Designer. The subreports that are included with your Mastercam installation use a naming convention that begins the filename with a hyphen. A few examples are listed below:

-Setup Sheet (MILL-FILE).rpx: Outputs a report section for each mill NC file selected by the user.

-Setup Sheet (MILL-OPERATION).rpx: Outputs a report section for each mill operation selected by the user.

-Setup Sheet (MILL-TOOL).rpx: Outputs a report section for each mill tool used.

-Setup Sheet (BLOCK).rpx: Outputs a report section for each block drill used.

If data associated with a subreport is not present in the XML file, that section of the report will not be output. For example, if an operation does not use a block drill, Mastercam does not write block drill data to the XML file. Since block drill data is not present in the XML file, the block drill subreport is not executed. The report will not contain a block drill section for that operation.
