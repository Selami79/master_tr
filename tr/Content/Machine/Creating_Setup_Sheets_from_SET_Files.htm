Creating Setup Sheets from .SET Files

The setup sheets you create using the .SET File program are automatically generated by the Setup Sheet function. They are based on a .SET file associated with the active machine group's control definition.

A .SET file is similar to a post processor file. It is a customization script that defines a specific setup sheet output. The .SET file can include:

  • format statements
  • processing logic
  • miscellaneous integers and custom variable definitions
  • system variables

Setup sheet output typically contains information on machine group operations and tools in the current part file. This output is formatted as ASCII text that opens in the Mastercam default editor. You can use your preferred text editor to view, edit, save, and print the setup sheet output file.

Working with default .SET templates

For each installed product, Mastercam provides a default metric and U.S. .SET template file. These files are located in the Mastercam installation location in the product's \POSTS subdirectory, for example ..\<Shared Data Folder>\mill\Posts. To create your own customized .SET files, copy and rename these templates (you can also rename a .PST file to a .SET format and modify accordingly), and then assign them to products and control definitions.

NOTE

All .SET files, including customized versions, must reside in the product-level \POSTS subdirectory.

Assigning .SET files

You can assign .SET files to a product, to the master version of a control definition stored in the Mastercam database, and to the local copy of a control definition stored in the Mastercam part file. The following hierarchy determines the .SET file that is used to create the setup sheet output.

  • Local Machine Group level: When working with a local copy of the machine and control definition in the Toolpaths Manager, edit the control definition associated with the machine group, assign a .SET file in the File usage list, and save the changes. This assignment is the most specific and overrides all other .SET file defaults.
  • Control Definition level: When creating control definitions in the Machine Type, Control Definition Manager, Files properties page, use the File usage list to assign a .SET file to a selected control definition. This assignment overrides the product level default. However, it is not used if you change the .SET file at the local machine group level as described above.
  • Product level: In the File, Configuration, Files properties page, File usage list, a .SET file is assigned to each installed Mastercam product.This is the most general default. The template .SET files provided with your installation are used as the initial defaults. However, you can change this assignment. If you use an external .NCI file to generate the setup sheet, the .SET file assigned to the machine type (product level) is used.
