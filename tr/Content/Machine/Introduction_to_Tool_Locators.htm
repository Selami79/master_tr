Introduction to Tool Locators

Mastercam Mill-Turn uses tool locators to mount tools on a turret or B-axis head. They connect the tool or tool assembly with the turret or tool spindle. Tool locators ensure that the tool is mounted in the correct orientation for the programmed toolpath, and allow Mastercam to accurately simulate your toolpaths. In the following picture, the dark gray components are the tool locators:

You can see tool locators if you go into the Tool Setup Manager. The number of tool locators that you see listed here is usually greater than the number of physical tool station blocks in your shop. This is because often, each block can be used by several different tool types, or tools can be mounted in it in different ways. Each such combination is defined in Mastercam as a different tool locator. For example, you might have one block in your shop that can be mounted differently depending on whether it is used with the left spindle or right spindle. In Mastercam, this results in two different tool locators even though the block itself is the same. So while it is easy to think of a tool locator as equivalent to a tool station or block, there is not a 1:1 relationship between them.

Mastercam loads your tools in the turret or tool spindle when you post your operations to the Sync Manager. Mastercam searches the component library for available tool locators and selects one that is appropriate for the tool and toolpath. You can also choose to load a tool manually by right-clicking on it in your toolpath settings and choosing Load tool on machine.

Once a tool has been mounted on a turret, you can see it in the Tool Setup Manager along with the tool locator.

Tool locator components

Tool locators are machine definition components. The tool locators for each .machine file are organized and stored in a component library with a .mcam-gmd extension. Each .machine file includes two libraries—one each for U.S. and metric tool locators.

The Tool Setup Manager lets you select and load tool locators from other component libraries in addition to the ones included with your .machine file—for example, a component library that you download from Mastercam's Tech Exchange.

Mastercam has different types of tool locators for different applications.

  • Turret-mounted locators for turning tools. These are used for basic stick tools as well as 3D tools, multi-insert tools, and Capto tools.

  • Turret-mounted locators for milling tools. These are used for live milling tools and include angled milling heads. The workflow for working with angled milling heads is different than for other tool locators; see Angled Milling Heads and Milling Tool Locators (Mill-Turn) to learn more.

  • Spindle-mounted locators for turning tools. These are used to mount a lathe turning tool in a tool spindle such as a B-axis head. The main benefit of doing this is that tools whose insert or cutting point is not on the spindle center line can be simulated accurately.

What is the difference between a tool locator and a tool holder. Aren't these the same?

These terms are used differently depending on the Mastercam product.

  • In Mastercam Lathe, the tool holder is typically the stick or shank on which the insert is mounted. Tool holders are defined in the Holders tab in the Lathe tool definition.

    Tool locators are not used in Mastercam Lathe—the holder connects directly to the turret. This is usually not a problem since Mastercam Lathe does not support Simulation.

  • In Mastercam Mill, a tool holder connects the tool to the tool spindle. It is defined on the Holder page in your Mill toolpath settings. Tool holders in Mill are revolved profiles that do not have a solid model associated with them. They are stored in a separate .holders file.

    Although tool locators are not used in Mill directly, when you create Mill toolpaths in Mill-Turn, you have access to both tool locators and the Holder page. In this case, use the tool locator to model the block that attaches to the turret, and the holder on the Holder page to model a sleeve or extension. If this is not needed for your application, create a very thin holder on the Holder page (for example, 0.001 inch).
