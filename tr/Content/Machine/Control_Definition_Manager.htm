Control Definition Manager

Use the Control Definition Manager for creating and editing control definitions. The control definition has two main functions:

  • It provides the post processor with the information it needs to properly process the toolpath data for the selected machine definition. This lets the post create an NC file which meets the control's requirements.
  • It tells Mastercam about the control and the post's capabilities and requirements so that users can't select options or features that the post won't support.

Control definitions are stored and organized in control files. Each control file can store several control definitions.

Identifying the control definition

Use the fields at the top of the dialog to identify the control definition. Each control definition is uniquely identified by its control type and post processor.

  • Use the Control type drop-down to select the type of machine this control is for. Only machine definitions of the same type will be able to use the control definition. You can also choose Existing definitions and select a control definition from the list to load an existing control definition from the current control file.

In general, when you are creating a new control, you should select the control type before you do anything else. If you select the control type after you enter any other data, Mastercam thinks you are changing the type of an existing control, instead of creating a new one. If you do change the control type, choose Yes when prompted to save the current work and continue with the editing session.

  • Use the Post Processor button to build the list of post processors that can be used with the control.
  • Enter an appropriate description in the Manufacturer and Description fields.
  • Select a post processor from the list. This is the post processor that you will work on in the current Control Definition Manager session. Mastercam displays a status icon next to each post in the list to tell you if it has been properly configured. If you select a post that was created for earlier versions of Mastercam, it will be automatically updated.
Organizing control definition data

The data stored in the control definition is organized into a number of topics, which are presented in the tree list on the left side of the dialog.

  • Click on a topic to see or edit the data it contains.
  • After a topic has been viewed, a green check mark will appear next to it.
  • Click Help when a topic is active to learn more about it.

The list of topics, and the content within each topic, will change depending on the selected Control type.

Opening and saving control definition files

Use the toolbar at the top of the dialog to:

  • Start creating a new control file.
  • Open an existing file.
  • Save the current file.
  • Save the current control definition to a control file.

Mastercam will not automatically save your data as you move from page to page. When you click Save, Mastercam automatically assigns a control extension to the file. Your editing and saving options will be different depending on how you enter the Control Definition Manager.

Parameters

New

Creates a blank control definition based on the current Control type. If there are unsaved changes in the current control definition session, you will be prompted to save them. If you entered from the Machine Definition Manager, the Control type will be preset to match the machine type.

Open

Opens a different control definition. If there are unsaved changes in the current control definition session, you will be prompted to save them.

Save

Writes the current data to the .mcam-control file on disk. This command writes the data from ALL the control definition topics, not just the one currently displayed.

Save as

Writes the current data to a different control definition file on disk. You can create a new name to create a new control definition, or select an existing file and overwrite it. This command writes the data from ALL the topics, not just the one currently displayed.

Save to machine group

Saves the current settings to the local copy of the control definition stored in the machine group. This does not affect the .mcam-control file, but will be saved with the part file and will be effective if the part file is opened on a different Mastercam workstation. This command writes the data from ALL the control definition topics, not just the one currently displayed.

Password settings

Opens the Control Definition Password Protection dialog box where you can enable or disable a password for your control definition.

Existing definitions

Click here to see a list of all the control definitions that are stored in the current .mcam-control file. You will be able to select one and load it into the Control Definition Manager for editing.

Control type

Select the type of machine tool attached to the control. You will only see options that you have been licensed for. If you entered from the Machine Definition Manager, you will be unable to select a machine type incompatible with the current machine definition.

Post processors: button

Opens a dialog box where you can manage the list of post processors that will be available for use with this control. This is the list of post processors that will be appear in the Machine Definition Manager when the machine definition is being created or edited.

Post processor: list

Select the post processor that you want to work on in the current Control Definition Manager session. Selecting a different post processor updates the current session with the Text values stored in the new post. Each entry in the list has a status icon. Selecting a post here does not change the post processor selection in the machine definition, which will be used in Mastercam for posting toolpaths. Use the Machine Definition Manager to select the post processor that will be used to post operations for that machine.

Manufacturer

Enter a plain-text description of the control manufacturer. The text you enter here will be applied to all the control definitions saved in the current .mcam-control file.

Description

Enter a plain-text description for the control definition. Unlike the Manufacturer field, this field applies only to the specific control definition displayed in the dialog box.

Control topics

Mastercam organizes the different control settings into topics so you can keep track of them easily. The list of topics that you see depends on the Control type. If you change the Control type, the list of topics will change to only those topics which are relevant to the selected type. Some topics, like Text, are further organized into subtopics which are visible when you click on the main topic.

Topics which have been viewed in the current editing session have a check mark next to them. Topics which have not been viewed have a question mark. The check marks do NOT indicate whether or not the settings for the topic have been changed or saved. Topics that have been locked display a lock icon.
