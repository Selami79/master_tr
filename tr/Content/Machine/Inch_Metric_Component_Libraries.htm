U.S. and Metric Component Libraries

Component libraries are typically supplied in both U.S. and metric units. If you are a developer whose .machine files will be distributed to many users, the best practice is to make each component library (.mcam-gmd file) available in both units.

Each time you add a new tool locator or other component to a library or make any other changes to one of the files, you should convert it to the other units and add both files to the .machine file.

Am I required to convert my component libraries?

No. For example, if you add a new tool locator to the metric component library because those are the units that you usually work in, Mastercam will not force you to convert it to U.S. units.

When Mastercam is loading tools and searching the component library for tool locators, it only searches the component library whose units match the active .config file. This means that if your desired tool locator is in the other component library, Mastercam will not see it.

However, you can always use the Tool Setup Manager to open the other component library and manually select a tool locator from it.

In the Machine Configuration page in Job Setup, you can also select a chuck or chuck jaws from either library, regardless of its units.

Follow these steps to convert a library to the other units.

  1. Click Save As and add _INCH or _MM to the file name—whichever units you are converting TO.
  2. Right-click the Machine Base component and select Convert units to inch or Convert units to metric.

  3. Save the library.
NOTE

Mastercam converts all of the settings for the component properties and scales the solid models associated with each component. However, any other entities in the file—such as construction geometry or pick points—are not converted.
