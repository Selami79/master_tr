Adding Posts, Machine, and Control Definitions

Mastercam uses the post processor, machine definition, and control definition to represent your machine tool and its capabilities. The machine definition is a model of your machine tool's capabilities and features and acts like a template for setting up machining jobs.

NOTE

If you want to update a machine definition, control definition, or post from a previous Mastercam version, refer to the Updating Files to the Current Version of Mastercam Help topic.

Machine Definition

The machine definitions in Mastercam are the starting point for creating machining jobs. Before you start a machining job in your shop, typically you first decide on what machine you will run the job, and then create the job based on that machine's features and capabilities. In Mastercam , you select a machine definition before creating any toolpaths. The machine definition is a model of your machine tool's capabilities and features and acts like a template for setting up machining jobs.

For more information, refer to the About the Machine Definition Help topic.

Creating a new machine definition

Before creating a new machine definition or editing an existing one, familiarize yourself with the Requirements for Basic Machine Definitions Help topic. All machine definitions need to meet minimum standards.

Then, refer to the Creating a Basic Machine Definition Help topic for step-by-step instructions on creating a basic machine definition.

Editing a machine definition

To edit an existing machine definition, refer to the Editing a Machine Definition Help topic. Mastercam lets you make changes to either the master copy of the machine definition or the local copy.

Mill-Turn machine files

When using Mastercam Mill-Turn, the *.machine file combines the machine definition from Mastercam Lathe with your Mill-Turn post, customizing your entire Mill-Turn experience for a specific machine tool.

For more information about the *.machine file, refer to the About the .machine File Help topic

Control Definition

Mastercam uses control files to store and organize control definitions. These are organized by machine type and post processor.

Each control definition consists of a machine type linked to a specific post processor. Each control definition inside a control file has a different combination of machine type and post processor. For more information about the control definition, refer to the How Mastercam Organizes Control Definitions Help topic.

To create a new control definition or make edits to an existing one, refer to the Creating and Editing a Control Definition Help topic.

Posts

Post processing, or posting, refers to the process by which the toolpaths in your part files are converted to a format that can be understood by your machine tool's control (for example, G-codes). A program called a post processor, or post, reads your part file and writes the appropriate NC code.

Generally, every machine tool or control requires its own post processor, customized to produce code formatted to meet its exact requirements.

For more information about post processing, refer to the About Post Processing Help topic. Additionally, refer to the Adding a Post to the Machine Definition Help topic for a how to add a post to a machine definition.
