MPBin Utility dialog box

Use the MPBin utility to encrypt all or part of a post processor (.PST or .MCPOST) or setup sheet (.SET) file. This lets Resellers and post developers protect their posts from being edited or viewed by unauthorized users. This is often referred to as "binning" a post.

Mastercam protects the post by:

  • Removing one or more blocks of code from the post file.
  • Saving the removed code in an encrypted binary file (.PSB or .MCBPOST file).
  • Keying the encrypted post to one or more license numbers and/or customer IDs.

In addition, you can password-protect the encrypted post to restrict updating with UpdatePost, or restrict use of a post to only users who have an active Maintenance plan.

Pay attention to the following notes:

  • MPBin can also be used to encrypt Machine Simulation (“MachSim”) machine definitions and related files.
  • You can select posts from any version of Mastercam. You will be prompted to update the posts if they are from an older version of Mastercam.
  • You are allowed to encrypt a post to an Evaluation license, but if the customer purchases the product, the license number will change and the post will need to be encrypted again. Similarly, if the Evaluation period is extended with a new license, the license number will change and the post will need to be encrypted again.
  • Customer IDs can only be used with software licenses, not HASPs or NetHASPs.

Carefully follow the steps in Preparing a Post for Encryption before running MPBin on your post.

Selecting an encryption method

There are three ways to encrypt the post.

  • Encrypt a post so that it can only be used by users with a specific type of license—for example, Industrial, Educational, or Reseller. Any user whose license matches the selected type will be able to run the post. Select the Encrypted file for any license of the specified type option.
  • Encrypt a post to one or more individual license numbers, where MPBin will create a different .PSB file for each individual license. Select the Separate encrypted files for specific serial numbers option.
  • Encrypt a post to one or more individual license numbers, where MPBin will create just one .PSB file that can be used by any of the specified licenses. Select the Single encrypted file for specific serial numbers option.
Re-encrypting a post

You can re-encrypt a .PSB file that has already been encrypted if its associated .PST file is available and it has been encrypted to a license type, not a license number. Simply insert new [STARTBIN] and [ENDBIN] keys in the .PST file.

If you are processing a .PSB file that has already been encrypted, you cannot encrypt to a type of license—for example, Industrial, Educational, or Reseller—if it was not selected when the post was originally encrypted. For example, if Educational was not selected when a post was originally encrypted, reprocessing the .PSB file and selecting Educational does not enable the new post to run on an Educational license. A message is generated in the log file indicating that the selection was disabled in the new post.

Deploying binned posts

Machine and control definitions always refer to the unbinned portion of the post (.PST or .MCPOST file). When creating a control definition or whenever you are prompted to select a post, always select the .PST or .MCPOST file.

The encrypted binary file (.PSB or .MCBPOST file) must always reside in the same folder as the .PST or .MCPOST file.

Managing your MPBin session

The MPBin utility is designed to make it easy for you to encrypt several posts to several different sets of users.

  • After selecting all of your encryption options, click Apply instead of OK to leave the MPBin utility open. You can then select new posts, new sets of license numbers, or both, and continue your session.
  • The dialog box for building lists of license numbers can also stay open. This lets you easily build different lists of licenses for the same set of posts.
  • The log file updates continuously throughout your MPBin session. Click Clear log file at any time to empty it.
Parameters

Input files

The list of files that will be encrypted. Even though only a single file is displayed in the field, you can select as many files as you wish.

Target directory

The folder where the encrypted files will be written.

Use the path of the input file as the target directory

Writes the encrypted files to the same folder as the input files.

Do not append license number to output file

By default MPBin appends the license number to the encrypted post. Select this option to prevent this. If more than one input file has been selected, this option will only apply to the first file.

Options

Select the encryption method from the following options:

  • The Encrypted file for any license of the specified type option encrypts the post so that it can only used by users with a specific type of license—for example, Industrial, Educational, or Reseller.
  • The Single encrypted file for all license numbers option encrypts a post to one or more license numbers and creates a single .PSB file that can be used by any of the licenses.
  • The Separate encrypted file for each license number option encrypts a post to one or more license numbers and creates a separate .PSB file for each individual license.

Always allow post to run on Reseller license

If this option is selected, the encrypted posts will always run on a workstation with a Reseller license. This means that no matter what other encryption options you select, any user with a Reseller license will be able to use the post.

Always allow post to run on Educational license

If this option is selected, the encrypted posts will always run on a workstation with an Educational license. This means that no matter what other encryption options you select, any user with an Educational license will be able to use the post.

Require active maintenance

If this option is selected, the encrypted posts will only run on a workstation with an active seat of Maintenance. This means that in addition to any other encryption options that you select, the user's workstation must have an active seat of Maintenance to use the post.

License type selection

Select the types of users who will be allowed to run the post based on their license type: Industrial, Educational, or Reseller. You can select more than one type of license.

Edit license number

Click this button to build the list of license numbers.

Timeout

Select the Enable timeout option to enter an expiration date for the post. Then click the Edit timeout button to enter the date.

Enable UpdatePost locking

Select the Enable UpdatePost locking option to prevent the post from being updated. Then click the Edit license number button to enter the license numbers of the users who will be authorized to update the post.

NOTE

Neither the UpdatePost utility nor the Migration Wizard will be able to update the post once it has been locked.

View log file

Opens the log file in your default text editor.

NOTE

The log file is always written to the \common\MPBin.log file, located in your shared Mastercam folder.

Clear log file

Deletes the current contents of the log file.
