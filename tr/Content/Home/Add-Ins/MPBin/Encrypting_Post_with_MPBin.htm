Encrypting a Post with MPBin

Follow these steps to encrypt a post:

  1. Make sure your original posts have been prepared with the proper [STARTBIN] and [ENDBIN] keywords, and that you have created a .PSM file. See Preparing a Post for Encryption to learn more.
  2. Choose Run Add-In from the Home tab or press [Alt+C].
  3. Select mpbin.dll.
  4. Click the Select button next to the Input files field and select the files to be encrypted. The names will appear in the Input files field.

    • These will typically be .PSM or .MCDPOST files if you have followed the instructions in Preparing a Post for Encryption.
    • You can also select .PST or .MCPOST files or setup sheets (.SET files).
    NOTE

    You can select posts from any version of Mastercam. However, Mastercam prompts you to run UpdatePost if they are from an earlier version of Mastercam.

  5. Click the Select button next to the Target directory field and select the folder where MPBin will write the encrypted posts.

  6. Choose either of two methods to allow access to the post:
    Allow access by type of license
    1. Select the Encrypted file for any license of the specified type option.
    2. Select the types of users who will be allowed to run the post. You can choose any or all of the license types:
      • Industrial
      • Educational
      • Reseller

    Allow access to specific license numbers or customer IDs only
    1. Choose from the following encryption methods:
      • Create one file that will be used by all users (Single encrypted file for all license numbers option)
      • Create separate files for each individual user (Separate encrypted file for each license number option).
    2. If desired, select the option to Always allow post to run on Reseller license.
    3. If desired, select the option to Always allow post to run on Educational license.
    4. If desired, select the option to Require active maintenance.
    5. Build the list of allowed license numbers and/or customer IDs. See MPBin License Entry dialog boxto learn more about building the list of licenses.

    MPBin supports both HASP/NetHASP numbers and software license numbers in addition to customer IDs.

  7. Users with a Reseller license can enter an expiration date for the post.
    1. Choose the Enable timeout option.
    2. Click the Edit timeout button to enter the date.
  8. Users with a Reseller license can choose to prevent the post from being updated.
    1. Choose the Enable UpdatePost locking option.
    2. Click the Edit license number button to enter the license numbers of the users who will be authorized to update the post. See MPBin UpdatePost Locking dialog box to learn more.
  9. Click OK to process the files and close the dialog box, or click Apply to process the selected files and leave the dialog box open. You can then select either a new set of posts, or a new set of license numbers.
  10. Click OK when prompted to review the results. MPBin writes the log file to the \common\MPBin.log file, located in your shared Mastercam folder.

Pay attention to these notes:

  • When removing text delimited by [STARTBIN] and [ENDBIN], MPBin leaves the [STARTBIN] line in the final .PST file. This [STARTBIN] tag marks where MP begins reading from the encrypted .PSB file.
  • If the post contains multiple [STARTBIN] and [ENDBIN] sets, MPBin combines the sections.
  • MPBin removes the [ENDBIN] line from the unencrypted post.
