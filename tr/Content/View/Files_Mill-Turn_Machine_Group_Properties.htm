Files tab: Machine Group Properties - Mill-Turn

Use the Files tab to manage the names and paths of files needed by the operations in a Mill-Turn machine group.

Toolpath directory

Displays the path where the NC file will be written. Choose the following button to select a different directory:

The default path is set in the .machine file.

Working with the .machine file

This tab displays the name of the .machine file that is loaded in the machine group.

  • Click Edit to edit the machine definition associated with the .machine file. Any changes that you make will be saved in the current part only.
  • Click Reload to reload the current .machine file. Do this when the .machine file may have changed and you want to ensure that your part is using the most current version. Expand the following topic to learn more.
    Reloading an updated .machine file

    Follow these steps.

    1. Make a backup copy of your current .machine file .
    2. Copy the new .machine file to the folder where your .machine files are stored. The default location is the \Mill Turn\machines folder; identify the proper location used in your shop.
    3. Open a part in Mastercam that uses the .machine file.
    4. In the Toolpaths Manager, choose Properties, Files.

      The Machine Group Properties dialog box opens with the Files tab displayed.

    5. Click the Reload button.

    Reloading the .machine file can result in serious changes to your part, depending on the changes that were made to the .machine file. You might need to do any or all of the following:

    • Regenerate one or more operations.
    • Delete and replace the part handling operations.
    • Reselect chucks and jaws.
    • Reload tools.

    In addition, any changes that you have made to the machine definition or other component files will be lost. These include:

    • New or edited reference or turret park positions.
    • Changes to tool definitions.
    • New or edited tool locators or other changes to the component library (.mcam-gmd file), including any additional tool locators that may have been imported from a different library.
    NOTE

    This procedure only works if you are reloading a .machine file with the same name. Only users with a Reseller or Developer license can replace one .machine file with another with a different name.

    Also, the updated .machine file must be from the same Mastercam version as the original .machine file and the part file.

Working with comments

Use the Group comment box to record comments that apply to all the operations in this group. Mastercam also lets you record operation-specific comments in the Toolpath parameters tab for each operation.

Parameters

Toolpath directory

Specifies where Mastercam stores the current machine group's NC programs.

Select path (button)

Click to select a new default path for the selected file type. All files of this type found in the selected directory will be listed in the File drop-down.

Group comment

Enter a comment that applies to the machine group.

Toolpath file name

Specifies the NC file name for all operations in the machine group. The default file name is set from the Toolpaths Manager page of the System Configuration dialog box.

Reload machine (button)

Select to reload the current .machine file.

Edit machine (button)

Opens the Machine Definition Manager to edit the machine definition. Any changes that you make apply to the current part only.

Operation library/defaults file

The Operation library/defaults file section displays the operations library and defaults file associated with the current machine definition.

How do I create or edit an operation defaults library (*.mcam-operations)?

Follow these steps to create a new operations library. This is a file with an .mcam-operations extension that contains commonly used toolpath operations that you can select and import into your current part file. Begin creating new operations files from a machine definition already loaded in the Toolpaths Manager.

  1. In the Toolpaths Manager, select Files under Properties.

  2. Select the Edit button to open the Edit Operation Library dialog box.

  3. Select Browse to open the Select Operation Defaults File dialog box.

  4. Right-click the operations file on which you want to base your new library, and choose Copy.

  5. Right-click in the window and choose Paste.

  6. Right-click the new copy of the operations file, choose Rename, enter the new file name, and press [Enter].

    NOTE

    When renaming your file, make sure not to change the file extension.

  7. Re-click the new file name. Make sure the new operations file name appears in the File name field.

  8. Click OK.

  9. If prompted to save changes to your operations library, click No.

  10. Edit the new operations library as necessary.

    NOTE

    You can also empty the new file and start with a blank file by choosing Delete all operations from the toolbar.

  11. When finished, click OK to save your changes and return to the Toolpaths Manager.

To edit an operations library, follow to step 4 and modify the operations in the list. You can do the following:

  • Edit the default parameters for a specific operation in the list by clicking Parameters.

  • Work in the operations list much like you can in the Toolpaths Manager. You can access many different options from the right-click menu, including adding, editing, and deleting operations. You can also perform other tasks similar to those in Toolpaths Manager such as selecting multiple operations, dragging and dropping operations within the list, and changing the names of operations.

Click OK when you are done. The operations library you just worked on is set as the current library for that machine group.

How do I create or edit toolpath default (*.mcam-defaults) files?

You can create new toolpath default files or edit existing ones. For more information about working with toolpath default files, refer to the Working with Toolpath Defaults Help topic.
