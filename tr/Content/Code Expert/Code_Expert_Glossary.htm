Code Expert Glossary

Term

Definition

AutoComplete

Code Expert's tool for automatically providing context-sensitive hints and prompts when you are typing in a code document.

Callback function, callback post block

A post block that is called and executed automatically when a specific event occurs.

Code Expert

An application introduced with Mastercam X7 that is designed to support Mastercam Mill-Turn. It provides an environment for a general-purpose text editor, Sync Manager, and Simulation. It is also used as the default editor for non-Mill-Turn Mastercam X7 and greater.

Consumer layer

A set of resources (tokens, formats, and post blocks) that you can edit when running in Consumer mode.

Control document

The set of layers in the .machine file that model the post and control definition. These include token definitions, post blocks, and formats. The control document appears in a tabbed window titled "Control." Each control document is divided into several layers.

Extensions

A window or shell that hosts a variety of new Mastercam applications, such as Code Expert and Tool Manager. While these are technically separate applications from Mastercam itself, behind-the-scenes channels provide communication with the main Mastercam application. The Extensions shell lets these new applications use a new, modern interface.

Format

A type of post resource that models a numeric format. In the Properties for each token, you can select a format that is applied when you output the token's value to the produced NC file.

IOF

Intermediate Output Format. This is the Mill-Turn equivalent of the ASCII NCI file used by Lathe MP posts. It contains toolpath motion within the context of the selected machine and other operation data that is read by an MP.NET post. It is also the native file format for Sync Manager documents, so files that you open in the Sync Manager have a .IOF extension.

Layer

A named set of tokens, post blocks, and formats that forms part of a .machine file. Access rights for different user roles are assigned per layer, so layers are typically used to expose or hide sets of resources to groups of users and to make it easy to import/export logical sets of resources.

Machine Explorer

The Code Expert window that lists all the components of the .machine file. These are organized in a tree.

Machine layer function

Special post blocks that map generic functions to machine-specific G-codes and M-codes.

MP.NET

The new object-oriented post processor language developed for Mastercam Mill-Turn.

Output window

The window that displays debug output and messages when you are debugging a post.

Part Handling file

An XML file that is used to configure the sequence of operations that are used by the Part Handling functions inside Mastercam.

Post Text file

An XML file that replicates the functionality of the post text section of a Lathe Mastercam post.

Resource

General term for tokens, formats, and post blocks. These are all listed in the Resource tree on the left side of Code Expert when you are working on a control document.

Sync Manager

The application within Code Expert that allows you to manage synchronization points between streams and visualize the toolpath information in tree and 3D graphical formats. Its user interface also allows you to set additional properties on the operations with respect to posted output (for example, enable G68.5 output). When Mastercam generates an IOF file from toolpaths and opens it in Code Expert, Sync Manager automatically display.

System layer

A set of resources supplied by Mastercam that defines a standard set of MP.NET resources common to all MP.NET machines. You cannot change these resources, but you typically override them via the Consumer layer to support specific machines.

Token

Tokens are like variables, but they have additional properties designed for NC applications like modality, multiple states, and formats. These properties are exposed in your post and can be easily accessed and manipulated.

Watch window

The window that displays the current states and values for all the attributes of selected tokens. Tokens need to be added to the watch list to appear in this window.
