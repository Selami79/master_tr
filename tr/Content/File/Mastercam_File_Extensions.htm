Mastercam File Extensions

Mastercam uses many types of data files, each of which is identified by its file name extension. With the release of Mastercam 2017, file extensions no longer contain version numbers. To ensure that your part, machine, and control definition files are not overwritten with new releases of Mastercam, the save file workflow has been revised.

Saving a part file

When you open a Mastercam X9 or older part file, clicking Save opens the Save As dialog box, where you can save the file in the .mcam format.

When you open a Mastercam 2017 or later part file in a newer version of Mastercam, you see a warning that the file is from a previous version of Mastercam. If you click Save, the Older Version File warning displays. You can choose to save the file to the current version of .mcam with the same name, or open the Save As dialog box where you can rename the file.

Saving a control definition file

When you open a Mastercam X9 or older control definition file, Mastercam checks if there is a file with the same name. If one exists, the Save As dialog box opens. If no matching file exists, the control definition file is saved to the current version of .mcam-control.

When you open a Mastercam 2017 or later control definition file in a newer version of Mastercam, the control definition warning message displays. You can choose to create a backup of the file and then update the file to the current version of .mcam-control with the same name, or you can open the Save As dialog box and rename the file.

Backup files are saved to the location of the old control definition file in the following folder (where XXXX is the original version number):

\Older Versions\XXXX

Saving a machine definition file

Any changes made to Mastercam X9 or older machine definition files are saved with their original extensions.

When you open a Mastercam 2017 or later machine definition file in a newer version of Mastercam and make a change, a backup of the original file is saved before updating the machine definition file to the current version.

Backup files are saved to the location of the old machine definition file in the following folder (where XXXX is the original version number):

\Older Versions\XXXX

Content files

Content files allow for easy installation of Mastercam specific files such as posts, tool libraries, and machine definitions. Double click or drag-and-drop the .mcam-content file into the graphics window. Mastercam extracts and installs the files in the proper locations. Content files are typically distributed by Mastercam Technical Support or your Mastercam Reseller.

For post processors, Mastercam supports the file name extensions as seen in the following table:

File Type Old Extension New Extension
Post file .pst .mcpost
Encrypted post file .psb .mcbpost
Source file for encrypted post .psm .mcdpost
Setup sheet .set .set (unchanged)

Changing your posts to use the new extension is a manual process. If you do change the extension of your .pst file, existing control definitions do not automatically recognize the new extension. For each renamed post, you must build a new control definition and import the settings from the old control definition.
