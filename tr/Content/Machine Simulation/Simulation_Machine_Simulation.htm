Machine Simulation dialog box

Use the Machine Simulation dialog box to set simulation, post, and machine definition options. You should set these options before running Machine Simulation.

NOTE

  • Mill-Turn operations within Sync Manager are simulated with Mastercam Simulator rather than Machine Simulation. To use Machine Simulation, select Run from Mastercam's Machine tab.

  • The Post Settings tab options are not about your Mastercam post, but rather about the post used to run the simulation.

Click a drop-down below for more information about each tab.

Simulation

Use the Simulation tab of the Machine Simulation dialog box to specify options before you run machine simulation.

Parameters

Machine

Select the machine to be simulated. Note that only machines in the \Shared Mastercam 2025\machine simulation\machsim folder are available.

Geometry

Specifies the geometry to be shown in the machine simulation.

  • All elements: Selects all surfaces and solids in the graphics window.
  • Selected elements: Returns you to the graphics window to choose specific surfaces or solids.
  • From level: Uses the geometry on a selected Mastercam level.
  • None: No geometry is shown.
  • Load From STL: Uses the geometry from the selected STL file.

Position

Allows you to translate the geometry a specified amount on each axis. You can enter the X, Y, and Z translation values manually or by selecting from the screen. Use Automatic to allow Machine Simulation to handle the position.

Fixture

Specifies the fixture geometry to be shown in the machine simulation. You can define the geometry as follows:

  • Selected elements: Returns you to the graphics window to choose specific surfaces or solids.
  • From level: Uses the geometry on a selected Mastercam level.
  • None: No geometry is shown.
  • Load From STL: Uses the geometry from the selected STL file.
  • Use Mastercam fixture definition: Uses the fixtures defined in the current Mastercam file. The fixtures are defined in either Machine Group Setup or the Simulator Options dialog box.

Stock

Specifies the stock geometry to be shown in the machine simulation. You can define the geometry as follows:

  • Do not use stock: No stock shown in the simulation.
  • Use Mastercam stock definition: Uses the fixtures defined in the current Mastercam file. The stock is defined in Machine Group Setup.
  • From level: Uses the geometry on a selected Mastercam level.
  • Load STL file: Uses the geometry from the selected STL file.

Simulation tolerance

Specifies the tolerance for the simulation's workpiece, collision control, and stock.

Stock to leave on target workpiece

Specifies an offset for collision detection.

Autostart

Specifies whether machine simulation starts in Run mode.

Process only provided moves

Instructs Machine Simulation to represent the motion using only the defined points.

Add intermediate positions as needed for simulation

Instructs Machine Simulation to add motion between defined points using the subsequent options.

TCP on (based on machine angles)

Creates linear interpolation based on maximum distance and maximum angle change values.

TCP off

Creates arc motion between the intermediate points.

Maximum distance

Sets the maximum length segment to create between interpolated points.

Maximum angle change

Sets the maximum angle difference between subsequent positions.

Use Mastercam settings

Specifies whether to inherit colors and shortcut definitions from Mastercam.

Move list display values

Specifies whether move-list coordinates should be displayed in absolute or relative coordinates.

Post Settings

Use the Post Settings tab to set up the post options before you run machine simulation. You can set axis pole handling, paint interpolation, or retract and rewind to stay within limits.

Parameters

Automatic angle pair (constantly adjusting for minimal travel)

Chooses the solution that creates the smallest change in the machine axis values compared with the previous move.

Keep angle pair (keep chosen solution as much as possible)

Attempts to keep the solution chosen for Select between the two solutions for startup for the entire toolpath posting.

Force head or table rotation instead of linear axis moves

Attempts to use the machine rotational axis as much as possible for machining instead of moving the translation axes. This setting applies to 4-axis, 5-axis table-table, 5-axis head-table, Mill-Turn, and special 6-axis machines.

Limit the travel of linear axis to the side of the starting position

Applies when Force head or table rotation instead of linear axis moves is selected. Attempts to force the translation axes (linear axes) to be within the selected starting position. This maybe be helpful if you have a toolpath that consists of linking moves between two or more of its sections.

Avoid rotation axis direction change

Applies when Force head or table rotation instead of linear axis moves is selected. Attempts to avoid changes to the rotation axis direction.

Find head or table fix position to stay within machine limits

Attempts to find a solution for the entire toolpath in such a way that it meets two conditions:

  • The entire toolpath can be simulated within the machine's limits.

  • The solution avoids moving the rotational axis.

This option attempts to move the rotational axis as little as possible and to keep them constant.

Select between the two solutions for start-up

Choose the posting solution. You can choose between the following options:

  • By naming it: Choose between set solutions based on the machine kinematics and move orientation. The example below demonstrates Solution 1.

  • By providing the 1st/2nd rotation axis angle: These options depend on your machine type in combination with the toolpath type.

    • If a machine has more axes than the toolpath uses (for example a 5-axis machine and a 3-axis toolpath), the entered axis angle represents the position in which the table remains for the entire posting.

    • If the number of machine axes match the toolpath (for example, a 5-axis machine and a 5-axis toolpath), the entered axis value is used as the preferred start angle for the selected rotation axis. The post automatically chooses the solution that brings the machine closest to the desired start angle.

  • By providing the translation axis direction: Defines the start position used by Force head or table rotation instead of linear axis moves. You can choose between Standard or Custom options.

    • Standard: Choose from the following predetermined options, relative to the machine coordinate system.

      • 1st linear axis (+)/1st linear axis (-): Typically the positive or negative X-axis.

      • 2nd linear axis (+)/1st linear axis (-): Typically the positive or negative Y-axis.

      • 3rd linear axis (+)/3rd linear axis (-): Typically the positive or negative Z-axis.

    • Custom: Define a direction by specifying a vector, relative to the workpiece coordinate system.

NOTE

The selected solution may not be respected for the entire solution. The solution may change depending on your chosen solution.

Behavior around pole

Use this option to handle cases where the tool direction is almost parallel or parallel with a rotary axis direction, which can result in an infinite number of valid solutions (referred to as a singularity or pole). Choose from the following:

  • Keep unchanged rotational axis angles: The posted moves do not handle pole singularities. The generated posted moves will not handle pole singularities and will simply provide posted moves. In certain scenarios, the rotation axes could flip and linear axes might jump.

  • Interpolate the rotation angles linearly: Linearly interpolates the machine axis values of the rotary axis that correspond to the points in the singularity to ensure the machine has movement through the pole. The linear distribution of rotary axis values is based on the distance between the tool tip positions.

    NOTE

    If the difference between the rotation axis values of two consecutive moves is bigger than the defined Maximum angle (under Interpolation), the post uses that value and adds additional moves in the pole area to ensure the Maximum angle value is met.

  • Use rotation angle around pole to stay within linear axis limits: This option is no longer supported.
  • Linear interpolation of rotation angle around pole: Assigns rotary axis values between two angles linearly.
  • Smooth interpolation of rotation angle around pole: Like linear interpolation, except the change of the rotary axis at the start angle and end angle are used to further improve the rotary axis's acceleration profile.

Trigger angle tolerance

The post handles pole areas specially, and, in general, the arbitrary axis value (for example, the C axis) is "frozen" when the tool axis orientation is almost parallel to the Z-axis. Pole Angle Tolerance is the limit between the tool axis orientation and another axis and is used to decide that the post is dealing with such a pole. To avoid machining errors, a smaller Pole angle tolerance is recommended. However, small values might generate many C-axis motions by small tool axis orientation changes.

Type

Specifies the type of interpolation to use. You can choose from the following:

  • By vector: The interpolation between the segment's two end points will be done the shortest way. The motion between the two vectors of the end point occurs in a flat plane.

  • By machine angle (RTCP): The interpolation between the segment's two end points will be done gradually, respecting the machine's kinematics. The motion between the two vectors of the end point does not occur in a flat plane.

  • By axis values (non-RTCP): The axis values are linearly interpolated. The tool can tilt and the tool tip does not follow the line between two moves.

Maximum distance (Resulted feed moves)

In point interpolation, specifies the maximum feed distance.

Maximum angle (Resulted feed moves)

In point interpolation, specifies the maximum rapid angle.

Maximum distance (Resulted rapid rate)

In point interpolation, specifies the maximum rapid distance.

Maximum angle (Resulted rapid rate)

In point interpolation, specifies the maximum feed angle.

Filter duplicate moves

Eliminates duplicate toolpath motion.

Disable MultiXPost adding additional moves

Prevents MultiXPost from adding additional moves.

Angle tolerance for using limits

Specifies the rotational machine-limits checking accuracy. For example, suppose a B axis is limited to between 0 and 90 degrees with a rough tolerance of 0.5 degrees. If the toolpath needs a start angle of either B = -1 or B = +90.5, which are the possible solutions, B = +90.5 is used because it is within 0 and 90 degrees +/- tolerance of 0.5.

Retract when consecutive feed moves angle change more than

Adds retract moves if the angle change between two consecutive feed moves is larger than the entered value (in degrees). This helps to avoid very fast tool motion during machining, which could cause problems with the part or the tool.

Retract and rewind to respect limits (preferable during linking)

When the tool reaches one of the limits on the rotational axis, the tool retracts to a safe area and rotates back to the initial limit.

Advanced

Opens the Retract and Rewind Advanced Settings dialog box to further customize the rewinds and retracts. You can select a single-step or multi-step retract, set how the rewind movement is calculated, and create lead in/out moves.

Allow solution change for minimum retracts and rewinds

5-axis toolpaths only. Allows Mastercam to find the best solution that reduces the number of retracts.

Machine Definition

Use the Machine Definition tab of the Machine Simulation dialog box to set up machine definition options. You should do this before running Machine Simulation.

Parameters

Post settings selection

Allows you to define a machine configuration (post selection) for MultiXPost. Each machine configuration contains a machine definition and, optionally, post settings parameters. Use the Add and Remove buttons to add a new machine configuration or remove a machine configuration.

Post settings ID

Defines a unique ID name for the selected machine configuration for posting. If there is only one machine configuration, this drop-down is empty. Any changes that are made when a machine configuration is selected are automatically stored as part of the post settings.

Post type

Enabled when Detection mode is set to Autodetect semiautomatic or Manual mode. Sets the post type for machine simulation. Choose between the following:

  • General: The default setting. Appropriate for most settings and most toolpath types, such as milling.

  • Turning: Used for turning toolpaths. The machine definition must contain a rotation axis where the workpiece/stock will be placed onto.

  • Contour machine: Used for 6-axis toolpaths, where the 6th axis is used for tool orientation.

Detection mode

Defines how the kinematic description should be taken from the machine's XML file. You can choose from the following:

  • Autodetect full: Takes all settings from the XML file.
  • Autodetect semiautomatic: Takes all settings from the XML file, but the spindle direction can be changed.
  • Manual mode: Initially reads the XML settings, but all kinematic settings can be changed independently from the XML file.

Mounting

Enabled when Detection mode is set to Autodetect semiautomatic or Manual mode. Opens the Mounting Settings dialog box, where you can define mount adapters.

Tools and workpieces positions

Opens the Tools and Workpieces Settings dialog box. These settings define the tool's initial position, direction, and orientation. The settings also define the initial location and orientation of the workpiece and stock.

Tools and workpieces parameters

The parameters located in the Tools and Workpieces Settings dialog box are defined below.

Add/Remove/Rename

Adds, removes, or renames elements from the list.

Simple matrix definition

Specifies the rotation or translation moves for the selected element.

Rotation angle

Specifies the rotation angle around each coordinate system axis.

Shift distance

Specifies the distance along each coordinate system axis.

Advanced matrix definition

Specifics the 4x4 matrix for the rotation and translation moves for the selected element.

NOTE

Changing any Simple matrix definition or Advanced matrix definition values automatically updates the other values. Occasionally, there may be circumstances (such as a gimbal lock) where the conversion between the Simple matrix definition values and the Advanced matrix definition values are not possible and will not be modified.

Tool ID / Workpiece ID

The Tool ID and Workpiece ID defines the connections between one tool and one workpiece used by the MultiXPost to determine the kinematic branches for posting. This is especially useful for machines with multiple tools and workpiece locations. When posting, select the combination that represents the locations of the tool and workpiece to properly calculate the machine axis values.

Rotation axis / Linear axis

Defines the machine kinematics. You define the number of axes, their movement direction, and position. These options are used by the MultiXPost algorithm and represent the machine configuration.

First axis vector / Second axis vector / Third axis vector

Defines rotation axis with a vector. The direction of the vector indicates the direction of rotation. For example, 0,0,1 indicates the rotation around the Z-axis going counterclockwise.

Base point

Defines the position of the rotation axis vector.

First axis direction / Second axis direction / Third axis direction

Defines the unit vector along which the linear axis moves. The vector direction provides the positive movement of the axis direction. For example, 0,0,1 indicates a linear axis along the Z-axis, with a positive value along Z+.

Minimum / Maximum

Sets the minimum and maximum axis limit values that can be used during posting to find solutions within the machine limits. The entered values should correspond with the Angle tolerance for using limits option in the Post Settings tab.

Name

Defines the axis name that is used when the calculated axis values are exported.

Type

Defines the axis position in the machine kinematic tree. Choose between the following:

  • Head: Use when the axis is on the machine tool branch. This means that the tool is attached on that axis, or at least in the kinematic branch of the tree.

  • Table: Use when the axis on the machine workpiece/stock branch. This means that the workpiece/stock is attached on that axis, or at least in kinematic branch of the tree.

Additional fixed axis/turret

Opens the Fixed Axis/Turret Settings dialog box to define fixed values for rotation and translation axes when defining machines, including lathe machines with turrets. Use the buttons in the dialog box to add, remove, or rename axes. Use the Fixed value parameters to specify the value used for that axis during posting.
