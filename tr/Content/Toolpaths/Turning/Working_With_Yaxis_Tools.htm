Working with Y-axis Tools

Mastercam Mill-Turn supports Y-axis turning for all turning toolpaths except custom thread and B-axis contour turning. This means that the cutting plane of the tool is parallel to the YZ plane, instead of XZ:

Follow the guidelines in this topic for creating Y-axis tools and using them in operations.

Creating Y-axis tools

Support for Y-axis turning begins with the tool definition. Tools that can be used for Y-axis turning are identified in the tool definition as part of the holder properties.

  • Select the Y-axis option when creating a new tool, and then continue defining the tool and assembly as you would with other tools. This is the only setting that specifically pertains to Y-axis tools.
  • You cannot edit an existing tool definition to select this option. The Y-axis option is only available when creating a new tool.
  • Y-axis turning is only supported for 3D tools. You cannot create wireframe or parametric tool definitions for Y-axis turning.

When creating Y-axis tools, you will rely heavily on some tool assembly options that are not often used with traditional tools.

  • Typically, you will need to use the Offset feature in the Setup page to make sure that the cutting plane is in the correct position.

    Expand the topic below to learn how to apply this setting.

    Defining the cutting plane for a Y-axis tool

    Use the Offset distance to properly locate the cutting plane. It typically represents the distance from the connection plane to the cutting plane. For example, this picture shows the connection plane for a common type of holder:

    Then, on the Setup page for the insert, set the Offset distance by selecting a point on the cutting plane of the insert:

Using Y-axis tools in operations

To create a Y-axis turning operation, select a turning operation and then select a Y-axis tool. Y-axis tools are identified in the interface with a distinctive icon.

Mastercam creates a set of planes that you can use to create your toolpath with the proper tool orientation and spindle origin. It also locks the B-axis to the 90-degree position; this is required for Y-axis turning.

When you select a Y-axis tool, Mastercam updates the interface so that it references Y and Z coordinates, instead of X and Z.

Use the Tool Angle dialog box to set the angular orientation of the tool. For most traditional X-axis tools, only the 0-degree and 180-degree orientations are used, but for Y-axis tools the Other field will be used frequently. Use it to orient the tool to any desired angular position that the machine is capable of.

Reference point and reference position functionality have been enhanced to support the Y-axis tool orientation. Coordinate positions in all three axes can now be specified for reference points, not just X and Z.

Also, the available approach and retract strategies have been enhanced to include Y-first strategies:

Some machines that are capable of Y-axis turning do not support CSS during Y-axis turning. If your machine does not support CSS for Y-axis turning, Mastercam will approximate the CSS by incrementing the spindle speed in steps as the diameter changes.

Y-axis turning and .machine files

Support for Y-axis turning is limited to .machine files that include the proper components and other settings. If you are using Mill-Turn .machine files that were created before Mastercam 2025, your Reseller or machine developer must enable this support. This is true even if your .machine file includes the proper axis components; migrating the .machine file to Mastercam 2025 is not sufficient.

NOTE

Y-axis turning is supported in Mill-Turn only, not Lathe.
